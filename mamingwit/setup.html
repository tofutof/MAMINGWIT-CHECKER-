MAMINGWIT CHECKER
SETUP & INSTALLATION GUIDE — PHISHING URL DETECTION SYSTEM
SYSTEM REQUIREMENTS

Prerequisites

  • XAMPP (Apache + MySQL + PHP 8.0+) installed and running
  • PHP 8.0 or higher with MySQLi extension
  • Modern web browser (Chrome, Firefox, Edge, Safari)
  • Internet access for Google Fonts (or offline — fonts will fallback gracefully)
PROJECT STRUCTURE

File Organization

mamingwit/ ← Project root (place in htdocs) ├── index.php ← Main dashboard interface ├── api.php ← AJAX request handler ├── database.sql ← Database schema & seed data ├── includes/ │ ├── db.php ← Database singleton class │ └── analyzer.php ← URL analysis engine (15+ rules) ├── css/ │ └── style.css ← SIEM dashboard theme └── js/ └── app.js ← Frontend application logic
STEP 01

Place Files in XAMPP

Copy the entire mamingwit folder into your XAMPP htdocs directory:

Windows:  C:\xampp\htdocs\mamingwit\
macOS:    /Applications/XAMPP/htdocs/mamingwit/
Linux:    /opt/lampp/htdocs/mamingwit/
STEP 02

Start XAMPP Services

Open the XAMPP Control Panel and start both services:

  • START Apache (web server)
  • START MySQL (database)

Verify both show green status indicators before proceeding.

STEP 03

Set Up the Database

Navigate to http://localhost/phpmyadmin in your browser. Then:

  • Click the SQL tab in phpMyAdmin
  • Open the database.sql file from the project
  • Paste the entire SQL content into the query box
  • Click Go to execute
⚠ This will create the database mamingwit_db with all required tables and seed data automatically.

Alternatively, use the Import tab: phpMyAdmin → Import → Choose database.sql → Go

STEP 04 (Optional)

Adjust Database Configuration

If your MySQL uses a password or different port, edit includes/db.php:

define('DB_HOST', 'localhost');
define('DB_USER', 'root');         // ← your MySQL username
define('DB_PASS', '');            // ← your MySQL password
define('DB_NAME', 'mamingwit_db');
define('DB_PORT', 3306);
STEP 05

Access the Application

Open your browser and navigate to:

http://localhost/mamingwit/

The MAMINGWIT CHECKER dashboard should load with the SIEM-style interface.

DATABASE REFERENCE

Tables Created

  • url_checks — Stores all scanned URLs, risk scores, and analysis results
  • community_reports — User-submitted phishing reports with deduplication
  • blacklist — Known phishing domain blacklist (seeded with 10 examples)
  • phishing_keywords — Weighted keyword library (25 keywords pre-seeded)
FEATURES OVERVIEW

System Capabilities

  • 15+ Detection Rules — HTTPS, IP, length, keywords, TLD, brand spoofing, redirects, and more
  • Visual Risk Meter — Animated SVG arc gauge with Low/Medium/High classification
  • URL History — MySQL-backed scan log with repeated URL detection and warnings
  • Community Reports — Crowd-sourced phishing reports with 24-hour IP deduplication
  • Phishing vs Legit Comparison — Educational reference panel for users
  • Technical Breakdown Table — Every rule evaluated with pass/fail/score
  • Dashboard Statistics — Live counters for total scans, high-risk, reports, repeated URLs
  • Responsive Design — Works on desktop, tablet, and mobile devices
  • SIEM-Inspired UI — Dark theme with cybersecurity professional aesthetics
